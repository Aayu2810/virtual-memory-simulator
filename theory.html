Virtual Memory Theory

What is Virtual Memory?

Virtual memory is a memory management technique that provides an "idealized abstraction of the storage resources" to processes. It separates logical memory from physical memory, allowing programs to use more memory than physically available.

Key Benefits:

  • Programs can be larger than physical RAM
  • Multiple programs can run simultaneously
  • Memory protection between processes
  • Efficient memory utilization

Paging

Paging is a memory management scheme that eliminates the need for contiguous allocation of physical memory. It divides virtual memory into fixed-size blocks called pages and physical memory into blocks called frames.

Page Replacement Algorithms

1. FIFO (First-In-First-Out)

Principle: Replace the oldest page in memory.

Advantages: Simple to implement, low overhead

Use Case: General-purpose systems where simplicity is preferred

2. LRU (Least Recently Used)

Principle: Replace the page that hasn't been used for the longest time.

Advantages: Good performance, considers temporal locality

Disadvantages: Requires tracking access times, more complex

Time Complexity: O(n) per operation

Use Case: Systems where recent access patterns are important

3. Optimal (BÃ©lÃ¡dy's Algorithm)

Principle: Replace the page that won't be used for the longest time in the future.

Advantages: Theoretical best performance, minimum page faults

Disadvantages: Requires future knowledge, not implementable in practice

Time Complexity: O(nÂ²) per operation

Use Case: Benchmark for comparing other algorithms

4. LFU (Least Frequently Used)

Principle: Replace the page with the lowest access count.

Advantages: Keeps frequently accessed pages in memory

Disadvantages: Old pages with high counts may never be replaced

Time Complexity: O(n) per operation

Use Case: Systems with clear access frequency patterns

Page Faults

A page fault occurs when a program tries to access a page that is not currently in physical memory.

Page Fault Handling Steps:

  1. Check if the page is valid (in the page table)
  2. If invalid, raise page fault interrupt
  3. Find a free frame or select a victim page to replace
  4. Load the required page from disk into the frame
  5. Update the page table
  6. Resume the instruction that caused the fault
